**测试用例**: 20 个刁钻问题，覆盖 12 个场景类别

## 二、改进措施与效果

### 2.1 修复分块丢失问题（最大改进）

**问题**：`_remove_table_regions()` 函数太激进，用表头关键词定位后删到下一个 `\n\n`，导致大量正文被误删。

**修复**：移除该函数调用，直接拼接原文和 Markdown 表格。

**效果**：之前 4 个问题完全检索不到内容（夜间飞行、降级、继续教育），修复后全部召回。

### 2.2 增大检索召回池

| 参数 | 之前 | 之后 |
|------|------|------|
| `_RECALL_MULTIPLIER` | 4 | 8 |
| `_OUTPUT_TOP_K` | 5 | 8 |
| `MAX_CONTEXT_TOKENS` | 4000 | 6000 |
| `top_k` (chat.py) | 5 | 8 |

### 2.3 优化 Prompt

新增三条引导规则：
- **条件调整规则**：要求 LLM 同时找到基础值和调整规则并计算
- **否定推理**：明确列出的条件之外的即为否定
- **版本对比**：以最新版本为准

## 五、剩余问题

### 5.1 测试框架改进
当前关键词匹配过于严格，需要：
- 支持语义相似度判断
- 区分"答案中的关键词"和"计算过程中的关键词"
- 允许同义词（"不能"/"不可以"/"不允许"）

### 5.2 OCR 暂时禁用
OCR 调用在后台任务中可能卡住（API 超时），已暂时禁用。后续需要：
- 添加超时控制
- 异步 OCR 不阻塞主流程

### 5.3 响应时间
平均 18.8 秒偏长，主要瓶颈：
- Query Rewriter LLM 调用 ~3s
- Embedding API ~1s
- Reranker API ~2s
- LLM 生成 ~10s

## 六、结论

经过优化，Nova-RAG 企业中文知识库准确率从 70% 提升到 **85%**，零幻觉。

**核心能力**：
- ✅ 表格数据提取（95%+）
- ✅ 枚举型查询（100%）
- ✅ 条件推理（80%，之前 50%）
- ✅ 否定推理（80%，之前 0%）
- ✅ 跨页面关联（60%，之前 0%）

**仍需改进**：
- 复杂多步推理
- 测试框架的语义匹配
- 响应时间优化
