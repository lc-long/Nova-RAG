## 三、失败原因深度分析

### 3.1 检索失败是主因 (4/6)

夜间飞行规定（4.2节）、资质降级恢复、继续教育要求这三块内容**完全未被检索到**。可能原因：

1. **分块问题**：这些内容可能被分到了不同的 chunk，与查询的语义关联不够强
2. **BM25 词表不匹配**：用户问"夜间飞行"，但文档中是"夜间飞行规定（V3.2修订）"
3. **向量相似度不够**：这些段落的 embedding 与查询的 cosine distance 可能超过了阈值

### 3.2 多步推理能力不足 (2/6)

风速降30%（120 × 0.7 = 84）和夜间高度降50%这类**需要两步推理**的问题，RAG系统无法完成。这是当前架构的根本限制：

- RAG 只做"检索→生成"，不做"检索→计算→生成"
- LLM 需要同时看到基准值和调整规则才能推理，但检索可能只召回了其中一个

### 3.3 检索范围问题

测试发现一个规律：**表格数据和列表数据检索效果好，连续段落文本检索效果差**。

| 内容类型 | 检索成功率 | 原因 |
|----------|-----------|------|
| 表格数据 | ✅ 高 | Markdown 格式，关键词密集 |
| 列表/条款 | ✅ 高 | 结构化，关键词明确 |
| 连续段落 | ❌ 低 | 可能被分块截断，关键词分散 |
| 跨段落关联 | ❌ 很低 | 需要召回多个 chunk 并关联 |

## 六、改进建议

### 高优先级
1. **检索召回率**：4/6 失败是因为内容未被召回。需要：
   - 检查分块策略，确保段落不被不合理截断
   - 增加初始召回池大小（当前 `_RECALL_MULTIPLIER=4`）
   - 考虑对长文档使用更大的 top_k

2. **跨段落推理**：风速规则、夜间高度规则等需要关联两个段落才能回答。可以：
   - 在 prompt 中引导 LLM 注意"如果上下文中有条件调整规则，请一并应用"
   - 或增加检索后的上下文拼接策略

### 中优先级
3. **中文 PDF 分块优化**：检查 pdfplumber 对中文的分块质量
4. **测试框架改进**：当前的关键词匹配过于严格，应支持语义相似度判断

---

## 七、结论

Nova-RAG 在**事实型查询**（表格数据、枚举、法规引用）上表现优秀，准确率接近 90%。
但在**条件推理**和**跨段落关联**场景下仍有明显短板，准确率约 50%。

**综合评估**：系统可用，但需要针对中文长文档的检索召回率做进一步优化。
