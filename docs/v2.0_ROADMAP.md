| # | 局限 | 严重程度 | 根因 |
|---|------|---------|------|
| 1 | 扫描版/复杂表格 PDF 解析质量差 | 🔴 高 | `pdfplumber` 对表格结构感知弱 |

## V2.0 核心升级方向

### 1. PDF 解析引擎升级：`pdfplumber` → `PyMuPDF (fitz)`

**现状**：`pdfplumber` 依赖文本流重建，表格识别能力弱。

**目标**：引入 `PyMuPDF`：
```python
import fitz  # PyMuPDF
doc = fitz.open("file.pdf")
for page in doc:
    # 提取表格为结构化数据
    tables = page.find_tables()
    for table in tables:
        markdown = table.to_markdown()
```

**收益**：
- 表格 → Markdown 自动转换，避免碎片化
- 支持扫描版 PDF（OCR 集成预留）
- 页面坐标可用于引用高亮

### 2. 表格 Chunk 策略优化

**现状**：表格被拆散成不相关的文本行。

**目标**：表格整表入库或按「标题行」为单位切分：
```
[Chunk]  DJI Mavic 3 飞行参数表
| 参数 | 数值 |
| 最大飞行速度 | 75.6 km/h (21 m/s) |
| 最长飞行时间 | 约 46 分钟 |
```

**实现**：在 `ParentChildChunker` 中增加 `TableChunker` 分支。

### 3. 多模态大模型接入

**现状**：仅文本检索（text-to-text）。

**目标**：扩展图片、图表理解：
- 图表截图 → MiniMax 视觉模型 → 描述文本 → 向量入库
- PDF 内嵌图片 OCR + 描述

**接口设计**：
```python
# 新增 /upload_multimodal 接口
@app.post("/upload_multimodal")
async def upload_multimodal(file: UploadFile):
    # 提取图片 + 文字，合并入库
```

### 4. 异步任务队列（替代轮询）

**现状**：Go → Python `/ingest` 无回调确认，前端轮询 `collection.count()` 判断就绪。

**目标**：引入轻量任务队列（如 Redis 或内置队列）：
```python
# Go 上传后拿到 task_id
# Python 处理完写入 Redis
# Go/WebSocket 订阅任务状态变更
```

### 5. Go → Python SSE 直连优化

**现状**：Python `requests.post(stream=True)` 直连 MiniMax 时有时不返回中间 chunk（仅 `done: true`）。通过 Go 代理绕过。

**目标**：诊断根因（是否为 HTTP 206 / chunked encoding 交互问题），修复后实现 Python 直连稳定流式。

### 6. Prompt 幻觉防御加固

**现状**：模型有时将未召回的内容误判为"未找到"（如 OSS-Q1 GPL-3 明明在 chunk 中）。

**目标**：在 system prompt 中强化：
```
如果召回的 chunk 包含相关内容，你必须引用并回答。
只有当召回 chunk 确实不包含任何相关信息时，才回答"未找到"。
```

## 版本演进路线图

```

  └─ 局限：PDF 解析、多模态

v2.0 (下一里程碑)
  ├─ P0: PyMuPDF 替换 pdfplumber（表格解析）
  ├─ P0: Prompt 幻觉防御加固
  ├─ P1: Go → Python 任务队列（去除轮询）
  ├─ P1: Python SSE 直连修复
  └─ P2: 多模态接入（图片描述向量）

v3.0 (远期)
  └─ 企业级：SSO、审计日志、多租户
```
